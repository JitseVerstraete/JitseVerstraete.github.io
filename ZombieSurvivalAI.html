Zombie Survival AI

Github Repository Download

Technologies

C++

Diciplines

AI Programming, Pathfinding, Decision making

Duration

~40 Hours

Description

In this project I wrote an AI that plays a zombie survival game with the goal of scoring as many points as possible. The game contains houses that contain items which the player can use to heal themselves (medkit), restore stamina (food) or shoot at enemies (pistol). The agent gains points by staying alive, hitting and killing enemies and picking up items. Points will be deducted for every shot the agent misses. The game gets progressively more difficult as time goes on, which will spawn more enemies as well as stronger variants (more health, faster...). Every now and the a "purge zone" will appear on the map which will kill the agent if it is caught inside when it detonates.

I used various different techniques to make the AI as performant as possible. First of all, i split up the play area into a grid and everytime the agent visits a time, i mark it as "explored" so it doesn't try to explore it again. This is how I make the agent look voor houses which is the only way to obtain items. Once a house has been found, the location is remembered to come back to later to check if any new items have spawned. Found items are also stored along with what type they are. If the agent needs a specific item it will go to the closest item of that type is remembers The agent constantly spins around in circles to make sure it can spot enemies in time and increase the chance of finding items. The "meta" decisions made by the agent is driven by a FSM (Finite State Machine). Every state defines a certain behavior like when to look for specific items, when to flee from enemies/purge zonesl, when to explore the map...
